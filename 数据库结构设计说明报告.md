# 数据库结构设计说明报告

## 1. 设计背景与目标

### 1.1 背景
基于对中医实践考试系统的分析，特别是考试模式和训练模式的功能需求，现有的数据库结构需要进一步完善，以满足以下需求：

1. 每个操作的得分需要单独设置字段
2. 在考试模式下访问时显示得分
3. 在训练模式时不显示得分

### 1.2 设计目标
- 优化数据库结构，提高数据存储和查询效率
- 为每个操作单独设置得分字段，确保评分的准确性和详细性
- 设计区分考试模式和训练模式的机制，满足不同模式下的显示需求
- 保持数据的一致性和完整性

## 2. 现有数据库结构分析

### 2.1 现有表结构
- **students表**：存储学生基本信息
- **questions表**：存储题目信息
- **scores表**：存储考试成绩记录

### 2.2 存在的问题
1. **成绩记录结构不够灵活**：当前scores表使用固定的question1-question5和对应的得分字段，无法适应不同数量的题目
2. **操作得分字段缺失**：没有为每个操作单独设置得分字段
3. **模式区分机制缺失**：无法区分考试模式和训练模式的记录

## 3. 改进后的数据库结构设计

### 3.1 整体架构
- **数据库类型**：MySQL
- **ORM框架**：Sequelize
- **表数量**：4个核心表

### 3.2 表结构详情

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| studentId | STRING(20) | 主键，非空，唯一 | 学号 |
| name | STRING(50) | 非空 | 学生姓名 |
| createdAt | DATE | 默认当前时间 | 创建时间 |
| updatedAt | DATE | 默认当前时间，更新时自动更新 | 更新时间 |

#### 3.2.2 questions表（题目表）
| 字段名 | 数据类型 | 约束 | 描述 |

| questionId | STRING(20) | 主键，非空，唯一 | 题目ID |
| question | TEXT | 非空 | 题目内容 |
| category1 | STRING(50) | 非空 | 一级分类 |
| category2 | STRING(50) | 可空 | 二级分类 |
| scoringCriteria | TEXT | 非空 | 评分标准 |
| answer | TEXT | 非空 | 标准答案 |
| createdAt | DATE | 默认当前时间 | 创建时间 |
| updatedAt | DATE | 默认当前时间，更新时自动更新 | 更新时间 |

| questionScore | INTEGER | 可空 | 题目总分 |

| updatedAt | DATE | 默认当前时间，更新时自动更新 | 更新时间 |

### 3.3 索引设计

#### 3.3.1 exam_sessions表索引
- `idx_student_id`：基于studentId字段
- `idx_exam_time`：基于examTime字段
- `idx_mode`：基于mode字段

#### 3.3.2 exam_records表索引
- `idx_session_id`：基于sessionId字段
- `idx_question_id`：基于questionId字段

### 3.4 关联关系
- **exam_sessions → students**：通过studentId外键关联
- **exam_records → exam_sessions**：通过sessionId外键关联
- **exam_records → questions**：通过questionId外键关联

## 4. 关键设计点说明

### 4.1 操作得分字段设计
- 在exam_records表中为每个操作单独设置得分字段（operation1Score至operation5Score），确保每个操作的得分都有独立的存储空间
- 支持最多5个操作的得分记录，可根据实际需求扩展

### 4.2 模式区分机制
- 在exam_sessions表中添加mode字段，用于区分考试模式（exam）和训练模式（training）
- 在查询时根据mode字段的值决定是否显示得分信息

### 4.3 成绩记录优化
- 将原有的scores表拆分为exam_sessions和exam_records两个表，实现更灵活的成绩记录结构
- exam_sessions表存储考试会话的基本信息
- exam_records表存储每个题目的详细得分信息

### 4.4 数据冗余设计
- 在exam_records表中冗余存储questionContent字段，减少查询时的表连接操作，提高查询效率

## 5. 实现建议

### 5.1 数据库迁移
1. 创建新的exam_sessions和exam_records表
2. 将现有scores表中的数据迁移到新表中
3. 验证数据迁移的正确性
4. 保留原scores表一段时间，确保系统稳定后再删除

### 5.2 代码适配
1. 更新后端API，适配新的数据库结构
2. 修改前端代码，在考试模式下显示得分信息，在训练模式下不显示
3. 确保数据的一致性和完整性

### 5.3 查询优化
1. 为常用查询字段创建适当的索引
2. 优化查询语句，减少表连接操作
3. 考虑使用缓存机制，提高查询性能

## 6. 总结

改进后的数据库结构设计解决了现有系统的问题，实现了以下目标：

1. **为每个操作单独设置得分字段**：在exam_records表中为每个操作设置了独立的得分字段
2. **区分考试模式和训练模式**：通过exam_sessions表中的mode字段实现模式区分
3. **优化成绩记录结构**：将原有的scores表拆分为exam_sessions和exam_records两个表，提高了系统的灵活性和可扩展性
4. **提高查询效率**：通过合理的索引设计和数据冗余，提高了查询性能

此设计方案能够满足中医实践考试系统的需求，为系统的稳定运行和功能扩展提供了良好的数据库支持。
